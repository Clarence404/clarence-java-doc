云计算概述

一、基本概念

1、云计算的定义

云计算是通过互联网按需提供计算资源（服务器、存储、数据库、网络、软件等）的服务模式，用户只需为实际使用量付费，无需自建和维护物理基础设施。

2、服务模型

┌─────────────────────────────────────────┐
│              SaaS（软件即服务）             │  Gmail、Office 365、Salesforce
│  你管：数据和使用                           │
├─────────────────────────────────────────┤
│              PaaS（平台即服务）             │  阿里云 EDAS、AWS Elastic Beanstalk
│  你管：应用代码 + 数据                       │
├─────────────────────────────────────────┤
│              IaaS（基础设施即服务）           │  AWS EC2、阿里云 ECS、Azure VM
│  你管：OS + 运行时 + 应用 + 数据              │
├─────────────────────────────────────────┤
│              物理机房（自建）                │
│  你管：一切                                │
└─────────────────────────────────────────┘
模型你管理的部分云厂商管理的部分典型场景
IaaSOS、中间件、应用、数据硬件、网络、虚拟化自定义服务器环境
PaaS应用代码、数据OS、运行时、中间件快速部署应用，无需关心环境
SaaS数据和使用一切基础设施和应用直接使用现成软件

3、部署模型

模型说明适用场景
公有云资源由云厂商拥有，多租户共享中小企业、互联网业务
私有云资源由单个组织独享，自建或托管金融、政府、高安全要求
混合云公有云 + 私有云互通核心数据私有，弹性扩展用公有云
多云同时使用多个云厂商避免厂商锁定、跨区域容灾

三、云原生 vs 传统架构

对比维度传统架构云原生架构
部署单元物理机 / 虚拟机容器（Docker + K8s）
扩缩容手动，分钟级自动，秒级
交付频率周/月级发布持续交付，每日多次
故障恢复人工介入自愈（自动重启/替换）
资源利用率低（峰值预留）高（按需弹性）
运维复杂度低（单体）高（需掌握 K8s 等）

四、选型维度

1、国内业务优先考虑

维度说明
网络合规数据出境合规要求，金融/医疗行业有强制规定
ICP 备案国内公网服务器必须备案，阿里云/腾讯云可直接办理
网络质量国内云在境内延迟更低，BGP 多线接入更稳定
技术支持中文支持、本地化文档、工单响应

2、国际业务优先考虑

维度说明
全球节点AWS/Azure 全球 Region 覆盖最广
生态成熟度AWS 服务数量最多，第三方集成最丰富
开源兼容GCP 对 K8s / TensorFlow 支持最原生
成本需综合计算流量费、实例费、支持计划

3、成本估算要点

  • 按需 vs 预留实例：预留 1-3 年可节省 40-70%
  • Spot / 抢占式实例：适合批处理任务，最高节省 90%，但随时可能被回收
  • 数据传输费：入站通常免费，出站按 GB 计费，跨 Region 尤其昂贵
  • 存储分层：冷数据迁移到低频存储（S3 Glacier / OSS 归档）可大幅降费
